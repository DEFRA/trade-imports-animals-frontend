## Working conventions / gotchas for this environment

- **Bash: one command per call** — no `&&`/`;`/pipes chaining unrelated commands, no `cd x && cmd`.
- **Never put backticks in a `git commit -m "…"` message** (shell command-substitution eats them) — use `git commit -F <file>` for multi-line messages.
- **Don't run bare `node`/`node -e`** (denied) — use an `npm run` script or a written `.js` file if you need JS.
- Use `~/` not literal `/Users/...` in Bash tool commands.
- Redirect long test runs to a file and read once rather than grepping streaming output.
- Verify subagent claims (esp. "it's green") yourself before trusting them.
