# PLAN — obligations-standalone-spike

Assembled from the ten per-package file plans, `DESIGN-DECISION.md`, `synthesis.json` (45 themes over the
514-obligation catalogue), `obligations-catalogue.json` and `parity-facts.json` (which answered the five
code-answerable parity questions a–e). Companion artefact: `coverage-matrix.json` — one row per catalogue
obligation id (514 rows) mapping id → scope → handling file(s) → verifying test.

Path convention: bare paths (`engine/evaluate.js`) are relative to
`prototypes/standalone/obligations-standalone-spike/`; e2e paths are repo-relative (`prototypes/e2e/...`).

## (a) Overview

**What is being built.** A fully self-contained fifth standalone spike (alongside spikes A–D) implementing
the _obligations paradigm_: a portable declarative obligations catalogue (`model/obligations.json`, 30
records — 29 user-submitted fields plus the system-handled `premium`; committed UUID ids, meaningful names,
open types, cardinality/indexedBy, constraints; NO scoping, mandate or presentation per obligations.md:196 —
DEF-2, SHAPE-27), a Container-tree Flow (`model/flow.json`) carrying all pinned journey copy, and a pure
two-evaluator engine: the **ObligationEvaluator** (`engine/evaluate.js` — the advertised IP: prune → name
view → named scope predicates → most-restrictive mandates → stacked authored reasons → per-fulfilment
states; deterministic, zero-I/O, never mints ids — EVAL-1..7, DEF-3) and the **JourneyEvaluator** as small
pure primitives (`flow-eval/` — four-status roll-up STATUS-2/15/20, deterministic navigation NAV-1..25,
journey state NAV-33..35). One side-effecting orchestrator (`orchestrator/`) canonicalises writes, mints
stable indexed fulfilment ids (graft 4, FULF-8/9), actively wipes data on scope exit (the Yes–No–Yes parity
mechanism, SHAPE-19/INDEX-21), runs the one in-process sync system handler (the quote, DEF-16/EVAL-8) and
re-evaluates to a fixed point per request (graft 6, EVAL-33..36, tagged as realising the unreviewed EVAL-35
sketch). Routes are plumbing over a 20-export contract barrel (`contract/index.js`, interrogation Level 1,
TOOL-21) plus a journey shell with cookie-carried journeyId (provisional FULF-2 vs STATUS-X1_5 settlement).

**Acceptance gate.** The three existing shared Playwright specs (`prototypes/e2e/task-list-with-linear-tasks.spec.js`,
`mandatory-fields.spec.js`, `invalidation.spec.js`) run unchanged against the new journey via one JOURNEYS
entry (TEST-5). Below them sit five browserless tiers (`tests/`): static reachability (TEST-1),
model↔template alignment walker (TEST-7..12), bounded-enumeration completability + dead-mandate detection
(TEST-3/13/14), cross-Flow evaluator equivalence against `model/skeleton-flow.json` (TEST-4/16..30 —
demonstrated structurally, not proven), and the flagship rename-survival proof (SHAPE-6/PERSIST-24).
Three NEW edge specs land live, asserting the ruled outcomes (graft 15, rulings recorded in §Rulings) — see §(d).

**Coverage.** Of the 514 catalogue obligations: **322 implement** (built fully on the parity path),
**133 reduce** (real mechanism, shrunk inputs — never faked; each reduction ledgered in the spike README),
**59 defer** (zero code, logged in the README defer log). 455 ids (implement + reduce) are claimed by at
least one planned file; the 59 handled as `deferred` all belong to the ten defer themes in §(e) — including the code-generation
pipeline stages (TOOL-1/3..9/16/26/27/29, theme 10). The layering meta-ids (DEF-1/ARCH-6/ARCH-12/
ARCH-X1_1/OPEN2-3) are reduces claimed by the engine/flow-split files that realise them — each row of
`coverage-matrix.json` names the files and verifying test.

**Scale.** 115 planned files across 12 packages (~9.3k lines incl. tests), everything ≤150 lines except
`engine/evaluate.js` (~170, accepted per graft 8) and the model JSON files (exempt). ESM, vitest, colocated
`*.test.js`, no new npm dependencies anywhere (no PBT framework, no cheerio in `tests/helpers/`).

**Honesty rails carried through every step:** provisional settlements labelled in code AND README (graft 2);
graft-9 comments on every bespoke bypass; `routes/guard.js` decision logic pure in `contract/guards.js` and
wired LAST, only after the three shared specs are green; the post-submit freeze is a documented deviation
from spike-a's real (unfrozen — parity ruling a) behaviour, unobserved by the shared specs and ruled in on
2026-07-02 (Rulings item 1).

## (b) Ordered build steps

Each step = one package, red-to-green (every file lands with its colocated test green before the next file),
with a verification command to run before moving on. Order respects the cross-package dependencies stated in
the package build notes (model → engine → lib → flow-eval → orchestrator → shell → contract → templates →
routes → test tiers → integration → e2e edge specs → docs).

> Note on registration timing: the e2e package's own notes prefer first-commit JOURNEYS registration
> (continuously-red gate). This plan follows the workflow's integration sequence instead — registration and
> the shared-spec run happen at Step 11 — so mid-build `npm run test:prototype` stays green for the other
> four spikes. The discipline is preserved by Step 11 running BEFORE `routes/guard.js` is wired.

**Step 1 — `model`** (files: `model/obligations.json`, `model/obligations.test.js`, `model/flow.json`,
`model/messages.en.json`, `model/skeleton-flow.json`, `model/flow.test.js`, `model/messages.test.js`).
Internal order: (1) mint the 30 UUIDs once with a throwaway script, paste, commit — ids frozen forever
(PERSIST-24); (2) `obligations.json` + its test; (3) COPY AUDIT — lift every heading/label/button/hint/error
string verbatim from spike-a source (parity-facts is the checklist, spike-a is ground truth); (4) `flow.json`

- test — per Rulings item 3, fullName is the ONLY page-hard mandate anywhere; the remaining parity
  `required_at_save` fields {email, registration, hadClaims, coverType, excessAmount, extras, ncdYears} are
  authored page-soft and enforced as engine-mandatory (block at CYA POST, never at save);
  (5) `messages.en.json` — the key list IS the reason-code contract `engine/reasons.js` must emit; (6)
  `skeleton-flow.json`, then `messages.test.js`.
  **Verify:** `npx vitest run prototypes/standalone/obligations-standalone-spike/model` — all three suites green.

**Step 2 — `engine`** (11 files). Order: `identifiers.js` → `reasons.js` (needs messages.en.json from Step 1;
the lockstep coverage test is the contract) → `scope/registry.js`, `mandates.js`, `prune.js` (independent
pure leaves) → `scope/journey-rules.js` (THE engine-mandatory set, parity ruling c) →
`scope/expressiveness.js` (unit-only demos, graft 5) → `scope/index.js` → `load-model.js` →
`evaluate.js` (extends the doc EvaluationResult with per-obligation/per-fulfilment `fulfilled` booleans —
the agreed flow-eval interface) → `index.js` barrel + smoke.
**Verify:** scoped vitest run over `engine/`; the load-model integration test wires the REAL model +
journeyScopeRegistry + companion types.

**Step 3 — `lib-validation-i18n`** (15 files). Order: `i18n/resolve.js` (fixture catalogue first, then bind
`model/messages.en.json`) → `validation/format-checks.js` → `save-check.js` → `field-errors.js` →
`validation/index.js` → `lib/fields/` bottom-up (input/block/choice views → registry → errors → index) →
`lib/quote/` (premium formula parity-pinned to the pound; `reference.js` = 'CI-' + 6 hex, needed by
contract/status AND submit).
**Verify:** scoped vitest run; the rank-14 no-leaked-codes gate (every engine reason code + validation code
resolves against messages.en.json) must pass — it is the build-time half of graft 7.

**Step 4 — `flow-eval`** (11 files). Order: `presents.js` + `applies-when.js` → `container-status.js`
(seven-row truth table) → structural nav (`first-applicable-page.js`, `first-page-presenting-obligation.js`)
→ status-consuming nav (`first-unfulfilled-page.js`, `next-after.js`) → `section-entry.js` →
`journey-state.js` → the two barrels. Provisional labels required in code: optional-only-page NA/Fulfilled
pick (STATUS-3 vs STATUS-11), NAV-33/34 email-gate counting, nextAfter status-filtered rule.
**Verify:** scoped vitest run; fixtures deliberately include non-car-insurance trees (generality rail,
OPEN1-X2_1).

**Step 5 — `orchestrator-store`** (8 files). Order: `orchestrator/fulfilment-ids.js` (graft-4 import-ban +
no-fresh-ids-in-evaluator-output tests need Steps 2/4 present) → `store/journey-repository.js` +
`store/index.js` → `orchestrator/apply-answers.js` (owns the `name__encodeURIComponent(fid)` form-name
convention) → `scope-exit-wipe.js` → `system-handlers.js` (quote fires on scope ENTRY during the pass, never
at submit — risk-4 pin) → `fixed-point.js` (EVAL-35 header tag pinned by test; iteration order evaluate →
adopt pruned → syncDerived → wipe → handlers) → `orchestrator/index.js` (risk-7 invariant: every mutating
path flows write → fixed point → save).
**Verify:** scoped vitest run incl. the Yes–No–Yes-at-the-seam and ?change=1-wipe tests.

**Step 6 — `journey-shell`** (5 of 6 files — `dump.js` waits for Step 9). Order: `journey/config.js`
(flow.json alignment test goes green immediately since Step 1 landed) → `paths.js` → `journey-context.js`
(THE isolation seam, cookie path-scoped to BASE) → `journey/index.js` → `hub-view.js` (graft 10: per-section
visibility, Get your quote always-visible-inert; doc-default NA hide/signpost branch unit-only).
**Verify:** scoped vitest run.

**Step 7 — `contract-guards`** (7 files). Order: `contract/status.js` (the evaluation object every other
file consumes — agree its exact key set first) → `navigation.js` → `view.js` (write `guards.js#isFrozen`
early or stub) → `mutation.js` (checkSave evaluated over payload-merged candidate fulfilments — the
excessAmount same-POST block) → `submit.js` (graft-3 distinguishable stale-recheck) → `guards.js` (pure
function + tests land now; `routes/guard.js` wiring waits for Step 11) → `index.js` (surface-pinning test:
exactly 20 exports).
**Verify:** scoped vitest run + the REPL smoke through the barrel alone.

**Step 8 — `templates`** (11 files). Order: `layout.njk` + `partials/error-summary.njk` +
`partials/fields.njk` FIRST (four of the six accessible-name traps pinned in fields tests: exact Yes/No
labels, single-'Yes' reveal, Day/Month/Year, no `required` anywhere) — then in shared-spec order: `start.njk`
→ `hub.njk` → `page.njk` → `claims-list.njk` + `claims-add.njk` → `quote-summary.njk` →
`check-your-answers.njk` ('Change recent claims' composition test) → `confirmation.njk`.
**Verify:** scoped vitest run (nunjucks Environment mirroring `src/config/nunjucks/nunjucks.js` roots).

**Step 9 — `routes`** (12 files) **+ `dump.js`**. Order: `routes.js` (shellRoutes only) →
`routes/test-server.js` → `routes/shell.js` → `routes/page.js` (pages in shared-spec order) →
`routes/claims.js` → `routes/endings/` (quote-summary → check-your-answers → submit → confirmation) →
`routes/model-endpoints.js` → `dump.js` (needs the live contract). `routes/guard.js` is WRITTEN and
unit-tested now but NOT wired into the route table — that is Step 11's final act.
**Verify:** scoped vitest run — the route specs drive full request → contract → template round trips via
test-server (headless Yes–No–Yes, GDS error round trips, half-empty quote pricing).

**Step 10 — `test-tiers`** (17 files). Order: fixtures (`tests/fixtures/**`) → `reachability.test.js`
(pure JSON) → `helpers/extract-inputs.js` + its test → `helpers/render-page.js` → `alignment-walker.test.js`
→ `helpers/enumerate-states.js` → `completability.test.js` → `helpers/script-runner.js` →
`flow-equivalence.test.js` (LAST of the dynamic tiers — TEST-29) → `rename-survival.test.js` →
`tests/README.md` (pinned-test registry finalised against actual file layouts). FIRST ACTION: dry-run one
file to confirm the repo-root vitest `--exclude ./tests/**` glob does not swallow the nested spike `tests/`
dir; if it does, rename the folder rather than fight the root config.
**Verify:** full scoped spike suite green.

**Step 11 — Integration + acceptance gate** (the workflow-mandated closing sequence):

1. Register the plugin in `prototypes/standalone/index.js` (import `obligationsSpike` from
   `standalone/obligations-standalone-spike/routes.js`, register alongside spikes A–D) + chooser link.
2. Add the JOURNEYS entry in `prototypes/e2e/journey.js` (id `standalone-obligations`, grouped path =
   `journey/config.js` BASE — the two MUST match).
3. Add to `package.json`:
   `"test:obligations-standalone-spike": "TZ=UTC vitest run prototypes/standalone/obligations-standalone-spike --no-coverage"`.
4. Run the scoped unit suite: `npm run test:obligations-standalone-spike` — green.
5. Run `npm run test:prototype` — the three shared specs must pass for 'standalone obligations' (and stay
   green for spikes A–D). Journey-spec retries on a fresh stack are known-flaky-then-green.
6. ONLY NOW wire `routes/guard.js` via `withGuard` in `routes.js` (post-submit freeze branch lands with it,
   ruled in — Rulings item 1); re-run steps 4 and 5 (guard.test.js's shared-spec sentinel
   makes this executable in CI thereafter).

**Step 12 — `e2e-specs` (new edge specs)**: add `prototypes/e2e/obligations/` — `obligations-journey.js`
helper, `post-submit-freeze.spec.js` (asserts the freeze, Outcome A), `early-cya-access.spec.js` (asserts
open access + soft prompts + stale-recheck, Outcome A), `mandate-composition.spec.js` (asserts fullName-only
page-hard, Outcome B) — all live tests asserting the ruled outcomes (Rulings items 1–3), discovered by the
existing testDir/testMatch config — and the folder `README.md` (spec → ruling mapping).
**Verify:** `npm run test:prototype` green including the three new specs.

**Step 13 — `docs`**: write the spike `README.md` LAST (it transcribes shipped reality: contract-table
export names, provisional-settlement labels, guards behaviour vs ruling (a), EVAL-35 tag, run scripts), then
execute its five-item grep checklist (every reduce theme has a ledger row; defer log complete and code-free;
contract table greps to real exports; each provisional entry names a file carrying the matching comment;
parity-ruling section matches `contract/guards.js` as shipped).
